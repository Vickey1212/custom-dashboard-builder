# 1. What This Task Is

You need to build a **Custom Dashboard Builder Web App**.

Users should be able to:

* Create **Customer Orders**
* Use those orders as **data**
* Build a **custom dashboard** using **drag-and-drop widgets**
* Configure widgets to show **analytics (charts, KPI, tables)**

Think of it like a **mini analytics platform** similar to:

* PowerBI
* Tableau
* Notion dashboards
* Metabase

---

# 2. Main Parts of the Application

There are **3 major sections**.

### 1️⃣ Dashboard (View Mode)

The main page where users **see their configured dashboard**.

Features:

* Date filter
* Shows configured widgets
* "Configure Dashboard" button

Default state:

```
No widgets
```

---

### 2️⃣ Dashboard Configuration (Edit Mode)

When user clicks **Configure Dashboard**, open the **builder page**.

Users can:

* Drag widgets
* Resize widgets
* Configure widgets
* Delete widgets
* Save layout

Widgets available:

```
Charts
- Bar
- Line
- Pie
- Area
- Scatter

Tables
- Table

KPIs
- KPI Card
```

Widgets are placed inside a **grid canvas**.

---

### 3️⃣ Customer Order Module

This is the **data source for the dashboard**.

Initially:

```
Customer Order table = empty
```

User must **create orders**.

Orders contain:

Customer info:

* First name
* Last name
* Email
* Phone
* Address
* City
* State
* Postal
* Country

Order info:

* Product
* Quantity
* Unit Price
* Total Amount (auto)
* Status
* Created by

# 3. Dashboard Widgets

There are **3 widget types**.

---

# KPI Widget

Shows a **single aggregated value**.

Example:

```
Total Revenue: $45000
Total Orders: 120
Average Order Value: $320
```

User config:

* Title
* Width / Height
* Metric (field)
* Aggregation

Aggregation options:

```
Sum
Average
Count
```

Example:

```
Metric: Total Amount
Aggregation: Sum
Result: $25000
```

---

# Chart Widgets

Charts include:

```
Bar
Line
Area
Scatter
```

User config:

* X axis
* Y axis
* Color
* Show labels

Example:

```
X: Product
Y: Total Amount
```

Output:

Revenue per product chart.

---

# Pie Chart

Shows **distribution**.

Example:

```
Orders by Status
Orders by Product
```

Options:

* Chart data
* Show legend

---

# Table Widget

Shows **raw order data**.

Options:

* Select columns
* Sort
* Pagination
* Filters
* Font size
* Header color

Example columns:

```
Customer Name
Product
Quantity
Total Amount
Status
```

---

# 4. Dashboard Grid System

The dashboard uses a **grid layout system**.

Desktop:

```
12 columns
```

Widgets have:

```
width = columns
height = rows
```

Example:

```
Chart width = 5
height = 5
```

---

# Responsive Behavior

### Desktop

```
12 column grid
```

### Tablet

```
8 column grid
```

If widget width > 8

```
Move to next row
```

---

### Mobile

```
4 column grid
```

Widgets stack vertically.

---

Options:

It filters **customer order data** used in widgets.
